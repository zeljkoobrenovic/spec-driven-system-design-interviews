# `docs/` — generated output (do not hand-edit)

Everything under `docs/<group>/` is **generated by `build.py`** and committed
so GitHub Pages can deploy from this folder. Do not edit these files by hand —
the next `build.py` run wipes and regenerates each `docs/<group>/`, discarding
any manual changes.

```
docs/
  <group>/              # one deployable site per publishable data/ group
    index.html          # copy of _templates/index.html (overview)
    interview.html      # copy of _templates/interview.html (explorer)
    interview.js
    overview.js
    styles.css
    node-types.json
    icons/
    data/
      index.json        # copy of data/<group>/index.json
      <id>/interview.json
```

## To change what's here

Edit the source, not the output:

- App shell / behavior / styling → [`../_templates/`](../_templates/README.md)
- Interview content → [`../data/`](../data/README.md)

Then run `python3 build.py` (or `python3 build.py <group>`) and commit the
regenerated `docs/`.

`build.py` only manages the `docs/<group>/` subdirectories — this `README.md`
at the `docs/` root is not generated and is safe to keep here.

See also: [`../README.md`](../README.md) (project overview).
